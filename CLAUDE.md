## Workflow

1. Inspect relevant files and `git status` before work.
2. Make a short plan before complex changes.
3. Preserve existing structure and user changes.
4. Do not implement unspecified domain policy. Add unresolved items to `docs/open-questions.md`.
5. Keep secrets out of files, logs, output, and API responses.
6. Run verification commands after changes.
7. If verification fails, fix the cause and rerun.
8. Commit and push only when explicitly requested.
