# Implement

입력된 작업(`$ARGUMENTS`)을 프로젝트 공통 규칙에 맞게 구현한다.

1. 요구사항을 완료 조건과 금지 범위로 나눈다.
2. `AGENTS.md`, `docs/architecture.md`, `docs/conventions.md`, `docs/definition-of-done.md`와 관련 코드·테스트를 확인한다.
3. `git status`로 기존 사용자 변경을 확인하고 보존한다.
4. 복잡한 변경이면 영향받는 모듈과 검증을 포함한 짧은 구현 계획을 세운다.
5. 기능명세서에 필요한 최소 범위만 구현하고 기존 모듈 경계를 지킨다.
6. 변경한 동작의 성공·실패 경로를 검증하는 테스트를 작성하거나 수정한다.
7. definition of done의 전체 검증을 실행한다. 실패하면 원인을 수정하고 다시 실행한다.
8. 변경 파일, 핵심 구현, 검증 결과, 실행하지 못한 항목과 남은 위험을 보고한다.

명시적 요청이 없다면 commit과 push는 수행하지 않는다.
