# Review

현재 변경을 수정 권한이 없는 독립 reviewer 관점으로 먼저 검토한다. 발견 사항은 심각도가 높은 순서로 파일과 근거를 제시한다.

- 요청사항이 빠졌거나 금지 범위를 벗어난 구현이 있는가?
- 런타임 버그, 회귀 또는 종료되지 않는 리소스 가능성이 있는가?
- 인증 우회, 입력 검증 누락, unsafe query 등 보안 문제가 있는가?
- 비밀번호, 토큰, Authorization, Cookie, DB URL이 코드·로그·응답·문서에 노출되는가?
- 운영 오류 처리에서 내부 정보가 노출되거나 예상 오류가 누락되는가?
- 핵심 성공·실패 경로를 검증하는 테스트가 부족한가?
- 현재 필요하지 않은 계층, 범용화, 의존성 또는 과도한 추상화가 있는가?
- README와 agent 문서의 명령·환경변수·구조가 실제 구현과 일치하는가?
- Docker의 네트워크, 포트, health check, 비-root 실행과 CI 명령에 문제가 있는가?

검토 후 가능한 범위에서 definition of done 검증을 실행한다. 문제가 없다면 발견 사항이 없다고 명시하고, 검증하지 못한 영역과 남은 위험을 별도로 보고한다. 명시적 요청 없이 commit이나 push를 수행하지 않는다.
